# Security Policy

## Reporting a vulnerability

If you find a security issue in 16 Eyes itself (the npm package, CLI, or skill
instructions — not a finding *produced by* an audit, which belongs in that audit's own
report instead), please report it privately rather than opening a public issue:

- Open a [GitHub Security Advisory](https://github.com/kigiela/16-eyes/security/advisories/new)
  (private, visible only to maintainers until resolved), or
- Open a regular issue at https://github.com/kigiela/16-eyes/issues if the issue isn't
  sensitive.

We'll acknowledge reports and aim to ship a fix before any public disclosure.

## Supported versions

Only the latest published npm version (`npm view 16-eyes version`) receives fixes —
this is a small open-source project without a formal LTS/backport policy.
